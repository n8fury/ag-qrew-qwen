--- Feature: Task Management ---

Change type:     New
Risk level:      High
Affected areas:  Tasks page, task API
Dependencies:    Authenticated session

SFDIPOT Coverage Map
  Structure:   Task list, task form, error messages, success messages
  Function:    List tasks, create task, update task, delete task
  Data:        Task title (valid, over-length, empty), done flag (true, false)
  Interfaces:  Task API (list, create, update, delete)
  Platform:    Desktop, Mobile
  Operations:  Session persistence, rate limiting
  Time:        Timeout mid-flow, async operations

Oracles (FEW HICCUPPS)
  Claims      — behaviour stated in the source doc / spec
  Comparable  — consistent with sibling features in the same product
  History     — consistent with how this flow behaved before the change
  User expectations — what a reasonable user would expect here
  World       — real-world facts (dates, currency, arithmetic) hold
  Purpose     — serves the stated business intent
  Statutes/Standards — validation, a11y, security norms

What MUST work (positive expected results)
  Scenario                              | Expected Result
  --------------------------------------|----------------------------------------
  List tasks                            | Current list of tasks is displayed with an accurate count
  Create task with valid title         | Task is added to the list, user stays on /tasks
  Update task with valid title         | Task is updated in the list, user stays on /tasks
  Delete task                           | Task is removed from the list, user stays on /tasks

What MUST be handled (negative expected results)
  Scenario                              | Expected Result
  --------------------------------------|----------------------------------------
  Create task with missing title       | Error message displayed, task not added, user stays on /tasks
  Create task with over-length title   | Error message displayed, task not added, user stays on /tasks
  Update task with over-length title   | Error message displayed, task not updated, user stays on /tasks
  Unauthorized access attempt          | Redirect to / (sign-in page)

Regression concerns (existing flows that must keep working)
  - None
