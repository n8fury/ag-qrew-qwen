TEST CASES — Tasks
==================
Sprint:   1
Module:   tasks
Source:   Task Management (P1, High risk)
Risk:     High

Business Context:
  This screen allows authenticated users to create, view, update, and delete personal tasks. It is the core productivity surface.
  User roles in scope: Regular User only (no sharing or permissions yet)
  Critical paths: Add new task → appears in list; mark complete → visual change + persistence; delete → immediate removal


TC-001
------
Title:          Verify that entering a new task title and clicking Add creates the task and displays it in the task list
Type:           Functional
Priority:       High
Preconditions:  - User is authenticated and on the /tasks page
  - Task list is initially empty or contains existing tasks
Steps:
  1. Navigate to http://localhost:3100/tasks
  2. Enter 'Buy groceries' in the New Task input field
  3. Click the Add button
Test Data:
  - Task title: Buy groceries
Expected:
  - Should display a new task card in the task list containing the text "Buy groceries"
  - Should display a checkbox next to the task title
  - Should display a Delete button next to the task title


TC-002
------
Title:          Verify that clicking the checkbox next to a task marks it as completed and updates its visual state
Type:           Functional
Priority:       High
Preconditions:  - User is authenticated and on the /tasks page
  - At least one incomplete task exists in the list
Steps:
  1. Navigate to http://localhost:3100/tasks
  2. Click the checkbox next to the task titled "Buy groceries"
Test Data:
  - N/A
Expected:
  - Should strike through the task title text "Buy groceries"
  - Should change the task card background to light gray
  - Should persist the completed state after page refresh


TC-003

Title:          Verify that submitting an empty task title displays an inline validation error

Priority:       High

Steps:

TC-004
------
Title:          Verify that attempting to edit a task with a title longer than 255 characters fails and displays an error
Type:           Negative
Priority:       Medium
Preconditions:  - User is authenticated and on the /tasks page
  - At least one task exists in the list
Steps:
  1. Navigate to http://localhost:3100/tasks
  2. Click the Edit button next to any task
  3. Replace the task title with a 256-character string
  4. Click Save
Test Data:
  - Task title: a256charlongtasktitlethatisexactly256characterslongandexceedsthemaximumallowedlength (256 chars)
Expected:
  - Should display a red error banner containing the text "Task title must be 255 characters or fewer"
  - Should retain the original task title in the edit field
