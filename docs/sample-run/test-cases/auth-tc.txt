TEST CASES — Authentication
===========================
Sprint:   1
Module:   auth
Source:   User Authentication (P1, High risk)
Risk:     High

Business Context:
  This screen enables users to securely access their personal task dashboard. It is the critical entry point for all user activity.
  User roles in scope: Regular User (no admin or read-only distinction yet)
  Critical paths: Valid login → dashboard redirect; invalid login → persistent error feedback; empty submission → inline validation


TC-001
------
Title:          Verify that entering valid credentials and clicking Sign In redirects to the dashboard and displays the Welcome message
Type:           Functional
Priority:       High
Preconditions:  - User account exists with email 'user@test.com' and password 'Passw0rd!'
  - User is on the /login page
Steps:
  1. Navigate to http://localhost:3100/login
  2. Enter 'user@test.com' in the Email field
  3. Enter 'Passw0rd!' in the Password field
  4. Click the Sign In button
Test Data:
  - Email: user@test.com
  - Password: Passw0rd!
Expected:
  - Should redirect to http://localhost:3100/dashboard
  - Should display a welcome message containing the text "Welcome, user@test.com"


TC-002

  2. Click the Sign Out button in the top-right navigation bar
Test Data:
  - N/A

TC-003
------
Title:          Verify that submitting invalid credentials displays an appropriate error message
Type:           Negative
Priority:       High
Preconditions:  - User is on the /login page
Steps:
  1. Navigate to http://localhost:3100/login
  2. Enter 'invalid@test.com' in the Email field
  3. Enter 'wrongpass' in the Password field
  4. Click the Sign In button
Test Data:
  - Email: invalid@test.com
  - Password: wrongpass
Expected:
  - Should display a red error banner at the top of the form containing the text "Invalid email or password"
  - Should retain the entered email and password values in their respective fields

TC-006
------
Title:          Verify that the sign-in form renders correctly with proper labels, placeholders, and active button state
Type:           UI
Priority:       Medium
Preconditions:  - User is on the /login page
Steps:
  1. Navigate to http://localhost:3100/login
  2. Observe the sign-in form layout and elements
Test Data:
  - N/A
Expected:

  - Should display label "Email" above the first input field with placeholder "Enter your email"
  - Should display label "Password" above the second input field with placeholder "Enter your password"
